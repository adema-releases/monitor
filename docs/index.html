Infraestructura operativa para crecimiento sostenible

Ubuntu 24.04 + Coolify + Adema Core

Adema Core es la capa operativa para estandarizar despliegue, monitoreo y continuidad de apps Django en una VM Ubuntu 24.04 con Coolify y PostgreSQL.

Procesos repetibles, seguridad por defecto, observabilidad util y recuperacion ante incidentes con backups verificables.

Quick Start: push -> nodo nuevo

Abrir repo oficial

Si tienes una VM Ubuntu 24.04 y Coolify listo, estos comandos dejan el monitor y el panel web operativos.

git clone https://github.com/adema-releases/monitor
cd monitor
sudo find . -type f -name "*.sh" -exec chmod 755 {} \;
sudo bash run_monitor.sh
sudo bash setup_web_panel.sh

Que hace: clona el repositorio, entra al proyecto, ejecuta el asistente de configuracion y luego instala el panel web con systemd.

Herramientas usadas:

  • git clone: descarga el codigo del repo.
  • bash: interpreta y ejecuta los scripts shell.
  • systemd: el panel se publica como servicio para arranque automatico.

Importante (primera ejecucion)

No es error si aparece el asistente interactivo con prompts como PROJECT_CODE [demo]: o CLUSTER_ID [CLUSTER-DEMO-01]:. Completa los datos (o presiona Enter para dejar defaults), guarda y luego continua con setup_web_panel.sh.

Valores base sugeridos para demo

  • PROJECT_CODE: demo
  • CLUSTER_ID: CLUSTER-DEMO-01
  • DB_NAME_PREFIX: demo_db
  • DB_USER_PREFIX: demo_user
  • BACKUP_REMOTE: tu remote rclone real (ej: r2:demo-backups)
  • BREVO_RECIPIENT / BREVO_SENDER: correos reales de alertas

Salida esperada

URL del panel + token para login

Precondicion

VM Ubuntu 24.04 con Docker, PostgreSQL y rclone

Validacion minima

bash monitor/status_snapshot.sh devuelve JSON valido

Guia de Nodo Nuevo (MVP)

1

Crear el nodo con un patron de nombre

Antes de tocar la VM, define un nombre estandar para el nodo. Recomendado: NODO_XXX (ej: NODO_001, NODO_MAD_01). Esto te simplifica inventario, alertas y crecimiento futuro.

Tip de estandarizacion: usa siempre el mismo patron para hostnames, etiquetas de proveedor y nombre interno en Coolify.
2

Abrir la VM y actualizar sistema base

Conectate por SSH y ejecuta la puesta a punto inicial de Ubuntu para tener paquetes y parches al dia.

sudo apt update && sudo apt upgrade -y
sudo apt autoremove -y && sudo apt autoclean -y

Que hace: actualiza indices de paquetes, instala parches pendientes y limpia paquetes/cache para reducir superficie de errores y consumo en disco.

Comandos:

  • apt update: refresca metadatos de repositorios.
  • apt upgrade: aplica actualizaciones disponibles.
  • apt autoremove/autoclean: elimina dependencias huerfanas y cache obsoleta.
3

Instalar dependencias base del nodo

Instala herramientas minimas para operar el stack y preparar backups/seguridad.

# Instalar motor oficial de Docker y Compose V2
curl -fsSL https://get.docker.com | sh
# Instalar resto de dependencias del stack Adema
sudo apt install -y postgresql postgresql-contrib rclone openssl git ufw fail2ban

Que hace: instala Docker Engine oficial (con Compose V2) para compatibilidad con despliegues modernos de Coolify, y luego instala dependencias operativas del nodo.

Componentes instalados:

  • get.docker.com: instala Docker Engine oficial sobre Ubuntu.
  • docker compose (Compose V2): plugin moderno de orquestacion requerido por stacks actuales.
  • postgresql: servidor de base de datos relacional.
  • postgresql-contrib: extensiones adicionales de PostgreSQL.
  • rclone: sincronizacion de backups a almacenamiento remoto.
  • openssl: utilidades criptograficas y generacion de tokens/certificados.
  • git: control de versiones y despliegue desde repositorio.
  • ufw: firewall simplificado sobre iptables/nftables.
  • fail2ban: bloqueo automatico de intentos de acceso maliciosos.
4

Instalar Coolify en el nodo

Instala Coolify y guarda de inmediato las credenciales iniciales en tu gestor de claves.

curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash

Que hace: descarga el script oficial de instalacion de Coolify y lo ejecuta con bash para aprovisionar el panel y sus servicios.

Herramientas:

  • curl -fsSL: descarga silenciosa con fallo explicito ante errores HTTP.
  • bash: ejecuta el script recibido por pipe.
  • Coolify: orquestacion self-hosted para despliegues y operaciones.

Acceso al panel (paso a paso)

  1. Identifica la IP publica del nodo en tu proveedor (o por SSH con curl -4 ifconfig.me).
  2. Abre en navegador la URL que informa el instalador de Coolify. En tu caso, usa http://IP_PUBLICA:3000.
  3. Si no responde, valida firewall/puerto: sudo ufw allow 3000/tcp y vuelve a probar.
  4. Dentro del panel: crea usuario administrador y guarda URL, email, password y token en tu gestor de claves.
5

Completar onboarding de Coolify (Server Type)

En la nueva version de Coolify, despues del login aparece el asistente inicial. Aqui defines donde se desplegaran apps y bases.

Que opcion elegir y por que

  • This Machine: despliega en el mismo servidor donde corre Coolify. Es la opcion correcta para tu nodo unico y para este runbook.
  • Remote Server: conecta otro servidor por SSH para separar control-plane y workloads.
  • Hetzner Cloud: aprovisionamiento directo para cuentas Hetzner.
# En el wizard de Coolify:
1) Seleccionar "This Machine"
2) Continuar con Deploy/Next
3) Click en "Create My First Project"

Con esto dejas listo el entorno base dentro de Coolify y ya puedes pasar a conectar repositorio y desplegar.

6

Clonar este repo y preparar variables/secretos

Con el nodo listo, clona el proyecto y ejecuta el asistente para generar .monitor.env y .monitor.secrets.

# 1. Crear el usuario de operaciones y su espacio de trabajo
sudo useradd -m -s /bin/bash adema
sudo usermod -aG sudo adema

# 2. Clonar el Framework en la ruta correcta (fuera de root)
sudo -u adema git clone https://github.com/adema-releases/monitor /home/adema/monitor
cd /home/adema/monitor

# 3. Asegurar permisos y lanzar configuracion/panel web
sudo find . -type f -name "*.sh" -exec chmod 755 {} \;
sudo bash run_monitor.sh
sudo bash setup_web_panel.sh

Que hace: crea un usuario operativo dedicado (adema), clona el framework fuera de root en /home/adema/monitor, ejecuta la configuracion inicial y publica el panel web.

Componentes:

  • useradd/usermod: crea el usuario de operaciones y lo agrega a sudo.
  • git clone (sudo -u adema): descarga el repo con propietario correcto en /home/adema/monitor.
  • run_monitor.sh: asistente CLI de configuracion y tareas diarias.
  • setup_web_panel.sh: crea servicio systemd, entorno Python y token del panel.

7

Enlazar tu repositorio en Coolify y validar

En Coolify: Sources -> Add Source (GitHub/GitLab) -> autorizar acceso al repo -> crear recurso de app Django. Luego valida salud del nodo.

bash monitor/status_snapshot.sh
sudo systemctl status adema-web-panel.service
sudo bash setup_cron.sh

Que hace: valida salud del nodo, confirma que el servicio web esta activo en systemd y programa tareas recurrentes en cron.

Herramientas:

  • status_snapshot.sh: resumen tecnico del estado operativo.
  • systemctl status: inspeccion de estado del servicio.
  • cron: automatizacion de backups/monitor/sentinel por horario.

Si todo esta OK, ya tienes un nodo estandarizado y repetible para alojar multiples proyectos Django.

Operaciones Diarias

  • Alta tenant: sudo bash monitor/create_tenant.sh cli001
  • Test DB: bash monitor/test_tenant_db.sh cli001
  • Backup: sudo bash monitor/backup_project.sh
  • Restore: sudo bash monitor/restore_tenant.sh cli001 YYYY-MM-DD archivo.sql.gz
  • Centinela RAM: sudo bash monitor/sentinel_ram.sh

Checklist de Produccion

  1. Token del panel guardado en ubicacion segura.
  2. Firewall restringiendo puerto 5000 o acceso por VPN.
  3. Cron activo para backup, monitor report y sentinel.
  4. Restore probado en tenant de staging.

Recuperar Token y Reinstalar Panel

Guia rapida para incidentes comunes del panel web de Adema Core: token perdido, token vencido o servicio inestable.

1) Ver token actual

sudo grep '^ADEMA_WEB_TOKEN=' /etc/adema/web_panel.env

2) Regenerar token

  • CLI: opcion 10) Regenerar token del panel web
sudo bash rotate_web_token.sh

3) Reinstalar/actualizar panel

  • CLI: opcion 11) Instalar/actualizar panel web
sudo bash setup_web_panel.sh

4) Reiniciar y validar estado

  • CLI: opcion 12) Reiniciar servicio panel web
  • CLI: opcion 13) Ver estado servicio panel web
sudo systemctl status adema-web-panel.service

Troubleshooting rapido

Unauthorized / token invalido

sudo grep '^ADEMA_WEB_TOKEN=' /etc/adema/web_panel.env
sudo bash rotate_web_token.sh

Panel caido o sin respuesta

sudo systemctl status adema-web-panel.service --no-pager
sudo journalctl -u adema-web-panel.service -n 120 --no-pager
sudo ss -lntp | grep ':5000'

Eliminar tenants desde el panel (Papelera)

El borrado esta protegido por doble confirmacion. Primero se mueve a papelera y recien ahi se puede eliminar de forma definitiva.

  1. En Gestion de Tenants, click en Mover a papelera.
  2. Ir a Papelera de Tenants y elegir Borrar definitivo.
  3. Escribir el CLIENT_ID exacto.
  4. Escribir la frase de confirmacion solicitada (default: BORRAR TENANT).
  5. Confirmar y seguir el job en Logs en tiempo real.
