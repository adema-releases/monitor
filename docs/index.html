Runbook de Nodo Adema Monitor

Guia operativa para provisionar un nodo nuevo, activar monitor + panel web y comenzar a gestionar tenants de forma segura.

Quick Start: push -> nodo nuevo

Abrir repo oficial

Si acabas de hacer push y quieres un bootstrap limpio, estos 3 pasos son el camino corto recomendado.

git clone https://github.com/adema-releases/monitor
cd monitor && sudo bash run_monitor.sh
sudo bash setup_web_panel.sh

Salida esperada

URL del panel + token ADEMA_WEB_TOKEN

Precondicion

Servidor Ubuntu con Docker, PostgreSQL y rclone

Validacion minima

bash monitor/status_snapshot.sh devuelve JSON valido

Guia de Nodo Nuevo (MVP)

1

Provision del servidor

Usa Ubuntu 24.04 LTS y asegura paquetes base para Docker, PostgreSQL y backups remotos.

sudo apt update && sudo apt install -y docker.io postgresql postgresql-contrib rclone curl openssl
2

Clonar y configurar monitor

Clona el repositorio y ejecuta el launcher para crear .monitor.env y secretos.

git clone https://github.com/adema-releases/monitor
cd monitor
sudo bash run_monitor.sh
3

Activar panel web seguro

Instala el puente Flask bajo usuario adema, con token y permisos sudo acotados.

sudo bash setup_web_panel.sh
4

Validar salud y operaciones

Verifica snapshot JSON, servicio web y estado del cron de backups/alertas.

bash monitor/status_snapshot.sh
sudo systemctl status adema-web-panel.service
sudo bash setup_cron.sh

Operaciones Diarias

  • Alta tenant: sudo bash monitor/create_tenant.sh cli001
  • Test DB: bash monitor/test_tenant_db.sh cli001
  • Backup: sudo bash monitor/backup_project.sh
  • Restore: sudo bash monitor/restore_tenant.sh cli001 YYYY-MM-DD archivo.sql.gz
  • Centinela RAM: sudo bash monitor/sentinel_ram.sh

Checklist de Produccion

  1. Token del panel guardado en ubicacion segura.
  2. Firewall restringiendo puerto 5000 o acceso por VPN.
  3. Cron activo para backup, monitor report y sentinel.
  4. Restore probado en tenant de staging.

Matriz de Operaciones Criticas
